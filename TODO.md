# 待办事项

## 权限系统优化

### 1. 简化权限编号
- 当前权限编号可能过于复杂
- 需要设计更简洁的编号方案

### 2. 提示有权限请求时带上请求内容
- 用户收到权限请求时，应该看到具体的请求内容
- 让用户明确知道批准的是什么操作

### 3. 指令可以切换权限模式
- 通过指令切换权限模式时，会替换新的 agent
- 需要实现权限模式切换命令

### 4. 自动批准模式的通知
- 切换到自动批准模式后，仍需通知用户批准了哪个权限
- 保持操作透明度

### 5. 权限白名单机制
- 调研是否需要权限白名单功能
- 某些操作可以预先加入白名单自动批准

### 6. 定制工具使用
- 支持自定义工具配置
- 灵活控制工具的使用权限

### 7. 指定恢复 session
- 支持通过指令恢复指定的 session
- 方便用户切换不同的会话上下文

### 8. 使用飞书卡片实现权限请求交互 ⭐
- 权限请求时发送交互式卡片（带"批准"/"拒绝"按钮）
- 接收卡片按钮回调事件
- 将用户决策传递给 Agent
- 批准/拒绝后更新卡片状态（显示已处理）
- 替代当前的命令行方式（/approve /deny）
